# Handoffs

Create a handoff whenever work changes hands, a session ends, or meaningful
design/implementation needs context. Name files `YYYY-MM-DD-HHMM-role.md` in
your local time.

```md
# Handoff — YYYY-MM-DD HH:MM — role

## Completed
-

## Current state
-

## Decisions made
- Link to the relevant entry in `../DECISIONS.md`.

## Next action
- One specific task, including its acceptance test.

## Risks / unanswered questions
-

## Files changed
-
```

Keep handoffs factual and under roughly 200 words. Update `../STATE.md` and
`../BACKLOG.md` alongside the handoff when appropriate.
